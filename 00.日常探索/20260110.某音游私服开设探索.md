其实我玩这个游戏比较少，但也算是我接触的第一个音乐游戏了。

这次的搭建其实就是个尝试，还有很多内容没有攻克，仅仅是搭建之后能进去的地步，毕竟也是第一次搞。

其实按理说应该用 Windows Server 才对，但是呢我是在 Vmware 里面搭建在宿主机测试，为了方便就直接上 Win 7 了. 但是经过其他腐竹提示，其实高版本的 Windows server 照样是可以搞的。
> 其实这些老游戏的服务器都很喜欢用 windows server，包括一些 MMO RPG，所以你经常能看到那种一键打包的服务端是 Vmware 的 windows 镜像，经常就是 windows xp 或者 windows 7

你需要以下资源
- Xxjam 1.8 服务端
- Xxjam 1.8 客户端
- Windows Server (高版本 Windows Server 亦可，当然这里我就直接用 win 7 了)
- Vmware（建议在虚拟机跑一遍流程，等都跑通了再上云）
- Microsoft SQL Server（2008 r 2是最低要求，之前的 sql server 2000 是对于 1.5 版本的数据库而言的, 否则无法导入。）

## 虚拟机设置
其实注意一点即可，为了方便后期连接，需要将虚拟机的网络改为桥接，并设置复制宿主机的网络状态。
![image.png](https://raw.githubusercontent.com/LarmaxHarkerX/Obsidian_Images/main/image/20260110030326980.png)

这样虚拟机的 ip 可以直接在宿主机用，省的造成不必要的通信问题。

剩下的内存以及核心数量就根据自己的机子情况来改就可以了，主要是 Windows 7 还是可以安装 VM tool 的，复制文件以及剪贴板共享比较方便。

如果是 server 的话，VMware 根目录下有一个名为 Windows. Iso 的文件，其实那就是 VMtool 的光盘镜像，不过装不装的上就看运气了，毕竟大家下载镜像的地方都不一样

## Sql server 的安装
Windows 7 的安装就跳过，可以选择 pe/iso 安装，或者直接用别人打包好的 Vmware 镜像。毕竟现在安装 Windows 7 以及激活还是比较费劲的。

这里主要说一下 Sql Server 的安装流程。

首先将 sql server 的 iso 挂载到虚拟机并点击 dvd 光驱，他会自己 autorun。

总的来讲 Sql server 2008 的安装加了一系列没啥必要的东西，他会检测你的系统环境。

需要注意记得安装或者打开. Net 3.5
