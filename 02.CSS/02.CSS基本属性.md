## 颜色
其实上一个文章并没有摆明了说 css 怎么写，不用管，照葫芦画瓢就可以。

关于颜色的话，基础就两个属性，文字颜色 color和背景颜色 background-color。

``` html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <p class="test">测试文本</p>
  </body>
</html>
```

```css
.test {
  color: red;
}
```

关于颜色，你可以输入很多不同的变种

比如直接写上颜色的英文名称，不过这个在实际开发中比较少用，即使 css 给出了上百个颜色预设，估计用的时候你也想不起来

其次就是 rgb、rgba，hsl，rgb ->hex.

一般就是直接取色然后输入就可以了，如果想了解为啥是 0-255，自行去了解数制转换。

Background-color 的用法是一样的，我们可以设置一个 100*100 px 的正方形然后给予颜色尝试。

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
<div class="test"></div>
  </body>
</html>
```

```css
.test {
  height: 100px;
  width: 100px;
  background-color: red;
}
```

## 字体
准确来讲不只是更改字体，而且包括文字的大小等

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
<p class="test">测试文本</p>
  </body>
</html>
```

```css
.test {   
    font-size: 20px; 
}
```

Font-family 用来设置这部分内容可以用什么字体，可以写多个。不过一般来讲都是从 Google fonts 找几个写上去。不过我们可以用微软雅黑来测试，可以设置多个字体，用逗号分开，谁在前面就优先用谁，没有就递推。

```css
.test {
  font-family: "微软雅黑", sans-serif;
}
```

或者是直接用 Google 的字体来托管，国内的话可以直接用阿里妈妈 iconfont的字体托管。

![image.png](https://raw.githubusercontent.com/LarmaxHarkerX/Obsidian_Images/main/image/20260118002434712.png)

```css
@import url("https://fonts.googleapis.com/css2?family=Gravitas+One&display=swap");
.test {
  font-family: "Gravitas One", sans-serif;
}
```

可选字体托管
[bunny](https://fonts.bunny.net/)
[iconfont](https://www.iconfont.cn/)
[Google fonts](https://fonts.google.com/)

> [!WARNING]
> @import 一定要写在 css 最前面，否则可能会被覆盖或者调用失败。
> 因为 CSS 引擎在解析文件时，必须先确定所有的外部引用（Import），然后才能计算层叠顺序（Cascade） 。如果放在中间，浏览器可能会先渲染一部分样式，然后再跳回来加载字体，导致页面出现“闪烁”。

## 盒子模型
我们要了解我们创建一个盒子之后会有什么。

分别是内间距 padding，边框 border和外间距 margin，对于我们产生影响的，只有内间距和边框。

假设我们设置了一个 div 是 100 px 长，假设这个时候我们设置 padding 为 10 px，border 为 5 px，那么这个盒子实际占据的空间就是 130 px ( $100+10*2+5*2$ )

$$\text{实际宽度} = \text{width} + (\text{padding} \times 2) + (\text{border} \times 2)$$

唯一的解决方案就是 border-box

这个会强制把内边距和边框往里收，占据内容区。
```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
<div class="test"></div>
<div class="test2"></div>
  </body>
</html>
```

```css
.test {
  width: 100px;
  height: 100px;
  background-color: red;
  padding: 10px;
  border: 5px solid black;
}

.test2 {
  width: 100px;
  height: 100px;
  background-color: blue;
  padding: 10px;
  border: 5px solid black;
  box-sizing: border-box;
}
```

![image.png](https://raw.githubusercontent.com/LarmaxHarkerX/Obsidian_Images/main/image/20260118004020805.png)

就是这么个东西，别听那帮老毕等讲的那么复杂。

当然偷懒的做法是给全局都用上 border-box
```css
* {
  box-sizing: border-box;
}
```

看情况使用，不要乱用。
