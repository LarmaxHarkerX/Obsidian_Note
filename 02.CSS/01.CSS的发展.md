其实我们通过前面的学习，我们可以知道，单纯的 html 只能显示内容，满屏幕都是文字，没有什么别的东西。也没有什么颜色变化，确实，几十年前的那帮人也是这么想的，要是能像 Microsoft Word 一样调整页面元素的位置，改变字体大小和颜色就好了。

当时的解决方案就是发明更多的 html 标签，来专门控制。比如 center 标签就是让内容居中，等等。但这样会有一个问题，就是我得边写边想我要用什么标签来控制样式，而且这种做法说白了会给页面加入很多不必要的东西，影响写作。所以这种方案很快就被弃用了，但是你依然可以这么些，但确实有点👎。

``` html
<center>
  <h1>
    <font color="blue" size="15">Heading</font>
  </h1>
</center>
```

可以尝试一下，但千万不要学，这玩意早就被废弃了。

所以人们就在想办法把这两个东西分离，该写内容些内容，样式我到另外的地方去写去。

转折点发生在 class 的发明。当时 class 其实不是给 css 用的，当时的 class 其实是标记点，比如我可以用 class="note" 来表示某个区块是 note，其实就是一个快速定位的东西。Css 利用了这个特性。

假设你给一个标题写上 class，你就可以直接在外部弄一个 css 文件并在头部引用。

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1 class="test">Heading</h1>
  </body>
</html>
```

``` css
.test {
  color: red;
  font-size: 60px;
  text-align: center;
}
```

后面再更新，就允许 style id class 这三个东西一起用了

| **特性**   | **Class (类)** | **ID (身份证)** | **Style (内联)** |
| -------- | ------------- | ------------ | -------------- |
| **复用性**  | 极高（全站通用）      | 无（页面唯一）      | 极低（跟着标签走）      |
| **优先级**  | 中             | 高（霸道）        | 最高（无敌）         |
| **主要用途** | **写样式的核心**    | 锚点、JS 定位     | 临时调试（不推荐）      |

但其实不是很推荐用 id，id 在一个页面里只能出现一次。

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1 id="test">Heading</h1>
  </body>
</html>
```

``` css
#test {
  color: red;
  font-size: 60px;
  text-align: center;
}
```

Style 的写法就是直接在标签里写，但其实不是很推荐，外部 CSS 文件浏览器是可以**缓存**的。你打开第一个页面加载了 CSS，后面 99 个页面直接从缓存里秒开。但你写在 Style 里的样式，浏览器每开一个页面都得重新‘读’一遍

``` html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
  </head>
  <body>
    <h1 style="color: red; font-size: 60px; text-align: center">Heading</h1>
  </body>
</html>
```

而且你会发现这样有点像在页面里拉屎。

除了加载慢，最要命的是不好改。如果你有 100 个页面，每个标题都写了内联 style，老板让你把红色改成紫色，你就得手动改 100 遍。这时候你就会发现，这种写法不仅是‘拉屎’，还是在给自己挖坑。

所以你能很清楚的知道 class 写法的好处，好改，而且可以复用。

所以还是推荐使用外部 css 来进行样式操作。

再到后面就有更多的新的 css 技术了，不过对于初学者先了解到这里即可。

> Id 写法其实很少用到 css 的写作里，因为他优先级和权力太大了，它更多的是被用在 js 脚本的定位而不是写样式

当然即使我们还没学 js，我们依旧可以用这个特性，id 在 html 其实还充当锚点标记的功能，你可以用这个写进 a 标签内，然后在文章的某个地方直接传送

标记写法
```html
<h2 id="test">test</h2>
```

锚点
``` html
<a href="#test">点我直达test</a>
```

## 拓展
还记得我们上个文章说的属性吗，属性同样可以被用来充当 css 选择。但其实这种我个人不是很推荐，有点像全局主题，容易出问题不好维护，但在某些情况下确实很好用，尤其是在当今的新兴 web 框架下。

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1 draggable="true">Heading</h1>
    <h1 draggable="false">Heading</h1>
  </body>
</html>
```

``` css
h1[draggable="true"] {
  color: blue;
}
h1[draggable="false"] {
  color: red;
}
```

我个人建议初学者不要大规模使用它来搞“大装修”。因为它和功能属性绑定得太死，万一以后你想改功能却忘了改样式，页面就会变得难以维护。它最适合出现的场景，是配合一些新兴框架去处理那些“变来变去”的状态。
