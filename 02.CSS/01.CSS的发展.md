其实我们通过前面的学习，我们可以知道，单纯的 html 只能显示内容，满屏幕都是文字，没有什么别的东西。也没有什么颜色变化，确实，几十年前的那帮人也是这么想的，要是能像 Microsoft Word 一样调整页面元素的位置，改变字体大小和颜色就好了。

当时的解决方案就是发明更多的 html 标签，来专门控制。比如 center 标签就是让内容居中，等等。但这样会有一个问题，就是我得边写边想我要用什么标签来控制样式，而且这种做法说白了会给页面加入很多不必要的东西，影响写作。所以这种方案很快就被弃用了，但是你依然可以这么些，但确实有点👎。

``` html
<center>
  <h1>
    <font color="blue" size="15">Heading</font>
  </h1>
</center>
```

可以尝试一下，但千万不要学，这玩意早就被废弃了。

所以人们就在想办法把这两个东西分离，该写内容些内容，样式我到另外的地方去写去。

转折点发生在 class 的发明。当时 class 其实不是给 css 用的，当时的 class 其实是标记点，比如我可以用 class="note" 来表示某个区块是 note，其实就是一个快速定位的东西。Css 利用了这个特性。

假设你给一个标题写上 class，你就可以直接在外部弄一个 css 文件并在头部引用。

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1 class="test">Heading</h1>
  </body>
</html>
```

``` css
.test {
  color: red;
  font-size: 60px;
  text-align: center;
}
```

后面再更新，就允许 style id class 这三个东西一起用了，但其实不是很推荐用 id，id 在一个页面里只能出现一次。

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1 id="test">Heading</h1>
  </body>
</html>
```

``` css
#test {
  color: red;
  font-size: 60px;
  text-align: center;
}
```

Style 的写法就是直接在标签里写，但其实不是很推荐，说白了，你都编译到一个 css 里面你就加载一次，你都写到页面里面每次加载都得重新解析渲染，造成资源浪费。

``` html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
  </head>
  <body>
    <h1 style="color: red; font-size: 60px; text-align: center">Heading</h1>
  </body>
</html>
```

而且你会发现这样有点像在页面里拉屎。

除了加载慢，最要命的是不好改。如果你有 100 个页面，每个标题都写了内联 style，老板让你把红色改成紫色，你就得手动改 100 遍。这时候你就会发现，这种写法不仅是‘拉屎’，还是在给自己挖坑。

所以你能很清楚的知道 class 写法的好处，好改，而且可以复用。

所以还是推荐使用外部 css 来进行样式操作。

再到后面就有更多的新的 css 技术了，不过对于初学者先了解到这里即可。

> Id 写法其实很少用到 css 的写作里，因为他优先级和权力太大了，它更多的是被用在 js 脚本的定位而不是写样式
