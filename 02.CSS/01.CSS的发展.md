其实并不会说的很细致，哦我们可以看到在前面我们写的 html 简直就是弱爆了，满屏幕都是文字，根本没有什么别的东西。也没有什么颜色变化，确实，几十年前的那帮人也是这么想的，要是能像 Microsoft Word 一样调整页面元素的位置，改变字体大小和颜色就好了。

当时的解决方案就是发明更多的 html 标签，来专门控制。比如 center 标签就是让内容居中，等等。但这样会有一个问题，就是我得边写边想我要用什么标签来控制样式，而且这种做法说白了会给页面加入很多不必要的东西，影响写作。所以这种方案很快就被弃用了，但是你依然可以这么些，但确实有点👎。

后面就简单了，就发展为像之前我们给标签写属性一样给标签加样式了，起码我该写啥就写啥，后面再加上就可以了。

但其实这还不是最优解决方案，因为这样其实还是会污染页面，想象一下你在标签里面写入了大量的样式属性。后面就发展为了我可以给某个元素起个代号，然后在另外一个文件给这个代号写样式。这样能最大化的减轻页面的污染，当然后面还发展出了更好的方案，不过初学者了解到这里即可。
